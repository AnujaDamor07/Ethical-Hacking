# 🔐 Phases of Penetration Testing & the Hacking Lifecycle

Penetration testing follows a structured process — identify weaknesses, exploit them, and document everything along the way. It's essentially a real-world attack simulation, carried out legally and ethically.

---

## 📌 1. Planning & Scoping (Pre-Engagement)

**🎯 Goal:** Nail down the objectives, scope, and rules of engagement before anything else happens.

**Key activities:**
- Understand the client's objectives, security policies, and compliance requirements
- Define what's in scope — systems, applications, networks, APIs, physical assets
- Set clear boundaries (IP ranges, domains, accounts) for testing
- Agree on rules of engagement — timeframe, allowed tools, escalation procedures
- Get written authorization, every time — this is what keeps the engagement legal

---

## 🕵️ 2. Reconnaissance (Information Gathering)

**🎯 Goal:** Build up intelligence on the target to spot potential attack paths.

### 🧊 Passive Recon (no direct interaction with the target)
- WHOIS lookups, DNS enumeration, social media research
- Pulling metadata out of public documents or websites (tools like FOCA)

### 🔥 Active Recon (direct interaction with the target)
- Port scanning (Nmap), service enumeration, banner grabbing
- Network mapping, OS fingerprinting, using tools like Netcat or Shodan

---

## 🛡️ 3. Vulnerability Analysis

**🎯 Goal:** Identify and weigh up weaknesses that could actually be exploited.

**Key activities:**
- Run vulnerability scans (Nessus, OpenVAS, Qualys)
- Look specifically for:
  - Misconfigurations
  - Outdated software
  - Weak credentials
  - Exposed services or APIs
- Prioritize findings using CVE IDs and CVSS scores

---

## 💥 4. Exploitation

**🎯 Goal:** Actually exploit what was found, either to gain unauthorized access or to demonstrate real impact.

**Key activities:**
- Run known or custom exploits — SQLi, XSS, buffer overflows, etc.
- Lean on tools like Metasploit, Burp Suite, sqlmap
- Attempt privilege escalation (working toward root/admin)
- Try bypassing firewalls or authentication mechanisms

---

## 🧬 5. Post-Exploitation

**🎯 Goal:** Figure out exactly how far the compromise actually goes, and what it means for the business.

**Key activities:**
- Establish persistence (backdoors, cron jobs, startup scripts)
- Move laterally across the network (pivoting)
- Pull out sensitive data — credentials, tokens, intellectual property
- Assess the real impact on confidentiality, integrity, and availability

---

## 📝 6. Reporting

**🎯 Goal:** Write up everything found, how it was exploited, and what should be done about it.

**Key activities:**
- Put together a technical report covering:
  - Vulnerabilities discovered
  - Exploitation methods used
  - Systems or data that were actually accessed
- Back it up with screenshots, logs, payloads, and other evidence
- Provide clear remediation guidance — patches, config changes, best practices
- Deliver both a technical write-up and an executive summary for non-technical stakeholders

---

## 🔁 7. Remediation Verification (Retesting)

**🎯 Goal:** Confirm the fixes actually worked.

**Key activities:**
- Retest to make sure patches and config changes were applied correctly
- Validate the original issues are resolved, and check nothing new broke in the process

---

### 🧭 The Flow, End to End

```
Planning → Reconnaissance → Vulnerability Analysis → Exploitation → Post-Exploitation → Reporting → (Optional) Retesting
```

---

## ⚔️ Malicious Hacking vs. Ethical Penetration Testing

The phases look almost identical on paper — the difference is authorization, intent, and what happens at the very end.

| Phase | Malicious Hacking | Ethical Penetration Testing |
|---|---|---|
| Reconnaissance | Info gathering (passive/active) | Reconnaissance |
| Scanning | Identify open ports/services | Vulnerability Analysis |
| Gaining Access | Exploit vulnerabilities | Exploitation |
| Maintaining Access | Install backdoors, pivot | Post-Exploitation |
| Covering Tracks | Delete logs, hide evidence | ❌ Never done — ethical testing leaves a clean, documented trail |

---

## 🛠️ Tools Commonly Used at Each Phase

| Phase | Tools |
|---|---|
| Reconnaissance | theHarvester, Maltego, Shodan, FOCA |
| Scanning & Analysis | Nmap, Nessus, OpenVAS, Nikto |
| Exploitation | Metasploit, sqlmap, Burp Suite, custom scripts |
| Post-Exploitation | Mimikatz, BloodHound, PowerView |
| Reporting | Dradis, Serpico, Markdown, screenshot tools |
