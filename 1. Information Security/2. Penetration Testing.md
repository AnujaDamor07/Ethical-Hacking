# 🔐 Penetration Testing (Pentesting)

**Penetration Testing** is a simulated cyberattack against a system, application, or network — designed to identify and exploit security vulnerabilities just like a real attacker would. The goal is to find and fix weaknesses **before** malicious hackers get the chance to exploit them.

It's an authorized, controlled process carried out by ethical hackers (penetration testers) to mimic real-world attack tactics and assess just how far an attacker could actually get.

---

## 🎯 Objectives of Penetration Testing

- 🔎 Identify security vulnerabilities that could realistically be exploited
- 🛡️ Test how effective current security controls and defenses actually are
- ⚠️ Assess the potential impact of a successful attack
- 📊 Validate overall security posture and incident-response readiness
- 📋 Support compliance with standards like PCI DSS, HIPAA, and ISO 27001

---

## 🔑 Why It Matters

- Catches vulnerabilities proactively, before attackers find them first
- Helps satisfy compliance requirements
- Validates security controls and patch management practices
- Tests how well an organization can detect and respond to an active incident
- Builds security awareness and informs better training

---

## ⚙️ The Pentesting Process

### 1. 🧠 Planning & Reconnaissance
Define scope, goals, and rules of engagement up front, then gather intelligence — both passively (WHOIS, DNS records, public info) and actively (probing IP ranges, identifying live infrastructure).

### 2. 🔍 Scanning
Use tooling to discover live hosts, open ports, and running services.

**Two main flavors:**
- 🌐 **Network scanning** — mapping out infrastructure
- 🧪 **Vulnerability scanning** — checking for known weaknesses

### 3. 🚪 Gaining Access (Exploitation)
Exploit whatever vulnerabilities were found to gain a foothold or escalate privileges.
*Examples: SQL injection, buffer overflow exploitation, password cracking.*

### 4. 🔐 Maintaining Access (Post-Exploitation)
Try to establish persistence — backdoors, rootkits, etc. — to simulate what a real long-term threat presence would look like. This phase also covers assessing impact, escalating privileges further, and pivoting deeper into the network.

### 5. 📝 Analysis & Reporting
Document every vulnerability found, the methods used to exploit it, and any sensitive data accessed along the way. Pair this with clear remediation recommendations and risk ratings.

### 6. 🔄 Remediation & Retesting
Apply patches or configuration fixes, then retest to confirm the issues have actually been resolved — not just patched on paper.

---

## 🔍 Pentesting Methodologies (Knowledge Level)

- ⚫ **Black Box** — the tester starts with zero internal knowledge, simulating a true external attacker
- ⚪ **White Box** — the tester has full visibility, including source code and architecture, enabling deep, thorough analysis
- ⚫⚪ **Gray Box** — the tester gets partial knowledge (e.g. valid credentials), simulating a semi-insider or compromised account

## 📦 Types of Penetration Testing — Overview

| Type | Focus Area |
|---|---|
| 🌍 **External Network** | Internet-facing infrastructure — firewalls, routers, public servers |
| 🏢 **Internal Network** | Simulated insider threats or compromised devices within the trusted zone |
| 🌐 **Web Application** | OWASP Top 10-style issues — XSS, SQLi, CSRF, insecure deserialization |
| 🔌 **API** | REST/SOAP API security — auth flaws, input validation, rate limiting, data leakage |
| 📱 **Mobile Application** | iOS/Android apps — insecure storage, weak session handling, poor encryption |
| 🖥️ **Desktop Application** | Locally installed software — buffer overflows, privilege escalation, weak permissions |
| 📶 **Wireless** | Wi-Fi networks — weak encryption, rogue access points, poor segmentation |
| 🌐 **IoT** | Smart/embedded devices — default credentials, missing firmware updates, insecure protocols |
| ☁️ **Cloud** | AWS/Azure/GCP environments — misconfigurations, weak IAM policies, exposed assets |
| 🎭 **Social Engineering** | Phishing, vishing, pretexting — testing human vulnerability, not just technical |
| 🏛️ **Physical Security** | Unauthorized facility access, badge cloning, tailgating |
| 🔴 **Red Teaming** | Full-scope, multi-vector simulated attack mimicking an Advanced Persistent Threat (APT) |

## 🔍 Types of Testing — In Detail

### 1. Network Penetration Testing

Covers both sides of the network boundary:
- **Internal testing** — simulates an insider threat or a compromised device already inside the trusted zone
- **External testing** — probes internet-facing systems like firewalls, routers, and public servers
- **Network + PCI segmentation testing** — confirms that network segmentation actually meets PCI DSS requirements for protecting cardholder data

### 2. Web Application Penetration Testing

Targets the classic OWASP Top 10 categories:
- SQL Injection (SQLi)
- Cross-Site Scripting (XSS)
- Cross-Site Request Forgery (CSRF)
- Insecure deserialization

Often paired with **web + API testing** to verify the backend integration is just as solid as the frontend.

### 3. API Penetration Testing

Focuses specifically on REST/SOAP API security:
- Broken authentication
- Input validation failures
- Rate-limiting gaps
- Data leakage risks

### 4. Mobile Application Penetration Testing

Targets iOS and Android apps, checking for:
- Insecure local data storage
- Weak session management
- Poor encryption practices

Often extended into **mobile + API** testing, since most mobile apps lean heavily on backend APIs.

### 5. Desktop Application Penetration Testing

Tests locally installed software on user endpoints, looking at:
- Buffer overflows
- Privilege escalation paths
- Insecure local file storage or overly loose permissions

### 6. Wireless Network Penetration Testing

Focuses on Wi-Fi infrastructure specifically:
- Weak encryption (WEP/WPA-era flaws)
- Rogue access points
- Inadequate network segmentation

### 7. IoT Penetration Testing

Targets smart and connected devices, evaluating:
- Default or hardcoded credentials
- Lack of firmware update mechanisms
- Insecure communication protocols

### 8. Red Teaming

A full-scope simulated attack designed to replicate an Advanced Persistent Threat (APT). Combines:
- Social engineering
- Technical exploitation
- Physical intrusion attempts

The goal here isn't just finding vulnerabilities — it's testing how well the organization's **blue team** actually detects and responds to a real, sustained attack.

### 9. Cloud Penetration Testing

Focuses on cloud environments — AWS, Azure, GCP — identifying:
- Misconfigured services
- Weak IAM policies
- Publicly exposed assets

Often includes dedicated **cloud configuration testing**, covering things like firewall rules and storage bucket permissions (e.g. S3).

### 10. Social Engineering & Phishing Testing

Simulates attacks aimed at people rather than systems:
- Email phishing
- Vishing (voice-based social engineering)
- Pretexting / impersonation

This assesses employee awareness and how effective security training actually is in practice.

### 11. Threat Modeling & Security Hardening

- **Threat modeling** — identifies likely attack vectors early, during the design phase, before a system is even built
- **Security hardening** — configures systems to shrink the attack surface and strip out unnecessary services

### 12. Physical Penetration Testing

Tests the physical security of facilities and hardware:
- Unauthorized access to restricted areas
- Badge cloning
- Tailgating

This evaluates how robust real-world physical access controls actually are — not just what's written in policy.

## 🧰 Common Penetration Testing Tools

- 💣 **Metasploit** — exploitation framework
- 🕸️ **Burp Suite** — web application security testing
- 🌐 **Nmap** — network scanning and enumeration
- 🔬 **Wireshark** — packet analysis
- 🔐 **John the Ripper / Hashcat** — password cracking
- 🧬 **sqlmap** — automated SQL injection testing

---

## ✅ Benefits of Penetration Testing

- 🧠 Finds vulnerabilities proactively, before attackers do
- 📌 Helps prioritize remediation based on real-world risk, not just theoretical severity
- 🛡️ Validates whether existing security measures actually work
- 📚 Supports compliance and regulatory requirements
- 🚨 Strengthens incident-response readiness

---

## 🔄 Vulnerability Assessment vs. Penetration Testing

| Feature | Vulnerability Assessment | Penetration Testing |
|---|---|---|
| 🎯 **Purpose** | Identify and list vulnerabilities | Actively exploit vulnerabilities |
| 🔎 **Depth** | Broad and largely automated | Deep, manual, and exploit-driven |
| ✅ **Risk verification** | Can include false positives | Confirms vulnerabilities are genuinely exploitable |
| 🧰 **Tools** | Mostly automated scanners | A mix of tools and manual technique |
| 🧠 **Skill level required** | Moderate | High — real ethical hacking expertise needed |
