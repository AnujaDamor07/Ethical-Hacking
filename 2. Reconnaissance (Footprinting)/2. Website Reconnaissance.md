# 🌐 Website Reconnaissance

Website reconnaissance is the process of gathering publicly available information about a target site — used heavily in **ethical hacking**, **penetration testing**, **SEO research**, and **competitive analysis**.

## 1️⃣ What It's For & Why It Matters

**Main goals:**
- Understanding a site's structure and underlying tech stack
- Spotting potential vulnerabilities
- Pulling SEO insights or analyzing a competitor
- Checking for compliance issues or accidental data leaks

---

## 2️⃣ What to Actually Collect

- **Basic info** — IP address, DNS records, WHOIS data, SSL certificate details
- **Metadata** — page titles, headers, JS libraries in use, the CMS powering the site
- **Exposed info** — leaked emails, usernames, hidden/forgotten paths
- **Archived content** — older versions of the site, pulled via tools like the Wayback Machine

---

## 3️⃣ Files Worth Checking First

| File | Purpose |
|---|---|
| `sitemap.html` | A human-readable index of the site |
| `sitemap.xml` | The machine-readable index search engines use |
| `robots.txt` | Tells crawlers which paths to avoid — often hints at what's "interesting" |

**Online sitemap tools:**
🔗 [xml-sitemaps.com](https://www.xml-sitemaps.com)

## 4️⃣ Manual Reconnaissance

Just browsing the target site by hand can surface a surprising amount:
- Employee names and email addresses
- Office addresses
- Hidden URLs tucked away in blog posts or comment sections

---

## 5️⃣ Spidering vs. Crawling

These terms get used interchangeably, but they're slightly different in practice:

| Feature | Spidering | Crawling |
|---|---|---|
| **Task** | Discovering links | Discovering links *and* analyzing their content |
| **Tools** | Burp Suite, OWASP ZAP, Arachni | ZAP, Burp's crawler, custom scripts |
| **Output** | A list of URLs and directories | URLs plus metadata plus actual content |
| **Main challenge** | JS-heavy sites, pages requiring auth | Dynamic content, keeping scope under control |

### Spidering Steps
1. Start from the homepage (your seed URL)
2. Follow every internal link you can find
3. Log every URL and parameter discovered
4. Stay disciplined about limiting this to in-scope pages

### Crawling Steps
1. Download each page
2. Parse the HTML and extract links
3. Collect metadata — headers, forms, embedded scripts

**Popular tools for this:**
- [xml-sitemaps.com](https://www.xml-sitemaps.com)
- [Screaming Frog SEO Spider](https://www.screamingfrog.co.uk/seo-spider/)
- [ConvertCSV URL Extractor](https://www.convertcsv.com/url-extractor.htm)

## 6️⃣ CLI Recon Tools

### 🔧 Katana — Fast Web Crawler

```bash
git clone https://github.com/projectdiscovery/katana.git
cd katana/cmd/katana && go build
katana -u https://example.com
```

### 🔧 Dirhunt — Directory Scanner

```bash
pip install dirhunt
dirhunt https://example.com
dirhunt https://example.com --to-file output.json
```

---

## 7️⃣ Downloading a Website Locally

### 🔹 `wget` (Linux)

```bash
wget -m https://example.com
```
> `-m` enables mirror mode — it recursively pulls down the whole site structure.

### 🔹 HTTrack

**On Linux:**
```bash
apt install httrack
httrack
```

**On Windows:**
[Download HTTrack](https://www.httrack.com/page/2/en/index.html) → launch it → New Project → enter the target URL → Start.
