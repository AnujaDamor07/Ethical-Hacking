# `fping` — Fast Ping Tool for Network Scanning

**`fping`** is a high-performance, command-line network monitoring utility designed to transmit **ICMP Echo Request** packets to **multiple hosts simultaneously**.

While the traditional `ping` command operates sequentially—waiting for a response or a timeout from one host before processing the next—`fping` employs a parallel, non-blocking polling architecture. This makes it significantly faster and more resource-efficient when performing live host discovery, scanning extensive subnets, or automating network availability checks.

---

### Key Features of `fping`

* **Asynchronous Parallelism:** Dispatches ICMP packets to multiple targets concurrently, optimizing subnet scanning speeds.
* **Dual-Stack Adaptability:** Fully supports both IPv4 and IPv6 network deployments.
* **File-Based Input Arrays:** Parses complex, newline-separated target inventories directly from a file for headless automation.
* **CIDR Range Comprehension:** Dynamically maps complete subnet ranges via standard CIDR notation (e.g., `/24`, `/16`).
* **Granular Output Formatting:** Allows precise filtering to isolate specific system responses (e.g., outputting *only* responsive nodes).
* **Script Optimization:** Returns clean status logs and standard exit codes that integrate seamlessly into Bash, Python, or Go automation pipelines.

### Common `fping` Commands & Options Reference

#### Show Help or Usage Information

Displays the compiled flag directory and sub-argument options supported by your system's installed binary version:

```bash
fping -h
# or
fping --help

```

---

#### Basic Target Checking

```bash
# Ping an isolated individual IP address
fping 192.168.129.80

# Probe a specific target domain name via DNS resolution
fping dns.google

# Force the network check to route strictly over IPv4
fping -4 www.google.com

# Force the network check to route strictly over IPv6
fping -6 www.google.com

# Target multiple standalone IP addresses concurrently in a single string
fping 192.168.129.80 192.168.129.31 192.168.129.61

```

---

#### Target Ingestion via File Input Arrays

When tracking larger networks, you can store your target IPs inside a plain-text configuration file.

**Create an input inventory file (e.g., `ip.txt`):**

```text
192.168.129.80
192.168.129.145
192.168.129.10
192.168.129.144  
192.168.129.31  
192.168.129.61  

```

**Execute the scan using the file flag (`-f`):**

```bash
fping -f ip.txt

```

> 🔬 **Traffic Analysis Tip:** > When debugging file-based scanning inside a protocol analyzer like Wireshark:
> * To capture live responses from responsive nodes, filter for: `icmp`
> * To track downstream routing lookups for unreachable local targets, filter for: `arp`
> 
>

#### Subnet Sweep Execution Patterns

```bash
# Scan a complete Class-C subnet via native internal list generation (-g)
fping -g 192.168.129.0/24

# Perform a subnet sweep and print execution statistics at termination
fping -g 192.168.129.0/24 -s

# Show only active, responsive hosts (-a) and suppress unreachable errors (2>/dev/null)
fping -g 192.168.129.0/24 -a 2>/dev/null

# Cleanly write discovered, responsive IP addresses directly to a text file
fping -g 192.168.129.0/24 -a 2>/dev/null > live-ip.txt

```

---

#### Advanced Timings and Payload Tuning

```bash
# Print numeric IP mappings instead of executing reverse DNS lookups
fping -n 8.8.8.8

# Sweep a network subnet utilizing an randomized byte payload string (-R)
fping -g 122.168.84.0/24 -a -R

# Throttle the packet interval delay (-i) to 10000 milliseconds (10 seconds) to bypass rate limits
fping -g 122.168.84.0/24 -a -R -i 10000

```

### Summary: Operational Flags Quick Reference

| Flag Syntax | Structural Argument Name | Primary Functional Purpose |
| --- | --- | --- |
| `-g` | `--generate` | Automatically populates an execution target array from a CIDR notation or range bounds. |
| `-f` | `--file` | Directs the binary to ingest a line-by-line collection of target hosts from an external text file. |
| `-a` | `--alive` | Filters terminal output to show **only** active, responsive hosts on the screen. |
| `-u` | `--unreach` | Filters terminal output to show **only** hosts that failed to respond to the probe. |
| `-s` | `--stats` | Outputs an operational summary block detailing target counts, packet metrics, and timing data upon termination. |
| `-i` | `--interval` | Adjusts the timing delay interval (in milliseconds) between consecutive outgoing packets. |
