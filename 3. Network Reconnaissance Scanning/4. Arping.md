# `arping` – ARP-Based Network Discovery Tool

**`arping`** is an essential maintenance and network discovery utility used to probe hosts on a local network by transmitting **Address Resolution Protocol (ARP)** requests directly at the **Data Link Layer (Layer 2)** of the OSI model. Unlike traditional `ping`, which relies on **ICMP** at the **Network Layer (Layer 3)**, `arping` operates lower in the stack. This gives it the unique capability to discover and track hosts even if they are heavily firewalled or configured to ignore standard ICMP echo requests.

---

## Key Features

* **Layer 2 Operations:** Functions completely at the Data Link Layer, communicating via hardware MAC addresses rather than logical routing boundaries.
* **Firewall Evasion:** Safely identifies live hosts that drop ICMP traffic, as operating systems must respond to local ARP requests to maintain network functionality.
* **Hardware Enumeration:** Discovers and validates the exact **MAC address** of the target along with the packet round-trip time ($RTT$).
* **Dynamic Request Targeting:** Commences mapping via a Layer 2 **broadcast ARP frame** (`FF:FF:FF:FF:FF:FF`); once an authoritative response is indexed, it dynamically switches to structured **unicast** frames to monitor the host.
* **Security Constraints:** Because it crafts raw network frames, executing `arping` requires administrative **root privileges** (`sudo`) on Linux systems.
* **Non-Routable Scope:** Restricted entirely to the **local subnet**. Because routers strip and re-encapsulate Layer 2 headers, ARP packets cannot pass beyond a local gateway.
* **Advanced Implementations:** Depending on the package variant installed (e.g., Thomas Habets' vs. iputils), features may include advanced source interface binding or pinging hosts directly by their hardware MAC address.

---

## Common Options

| Option | Operational Description |
| --- | --- |
| `-c <count>` | Stop after sending a fixed number of ARP request packets. |
| `-f` | Finish and exit immediately after receiving the very first authoritative reply. |
| `-w <seconds>` | Enforce a hard timeout duration for the operation. |
| `-i <interface>` | Explicitly bind traffic to a specific network interface (e.g., `eth0`, `wlan0`). |
| `-q` | Quiet mode; suppresses standard terminal output (ideal for parsing in automation scripts). |
| `-v` | Verbose mode; prints detailed logging information and structural frame attributes. |
| `-b` | Force the utility to maintain only broadcast requests rather than switching to unicast. |
| `-D` | **Duplicate Address Detection** mode; returns exit status 0 if the targeted IP is unassigned (used to check for IP conflicts). |

---

## Why Use `arping`?

1. **Link-Layer Verification:** Confirms that local network interfaces and layer-2 switches are communicating correctly.
2. **Exposing ICMP Blocks:** Provides a highly reliable tool for network discovery inside subnets where local firewalls silently drop standard pings.
3. **MAC-to-IP Binding Inventory:** Quickly maps an IP address back to a physical network interface card (NIC) manufacturer signature.
4. **IP Conflict Resolution:** Easily diagnoses situations where two hardware endpoints are accidentally assigned the identical IP address on the subnet.

## Common `arping` Commands (Corrected & Structured)

### 1. Send Continuous ARP Requests to a Target

Continuously streams ARP broadcast frames targeting an IP on the local subnet until manually broken with `Ctrl+C`.

```bash
sudo arping 192.168.129.10

```

### 2. Send a Single Fixed-Count ARP Request

Transmits exactly **one ARP request frame** and prints the result. This is highly efficient for targeted availability checks or scripting conditions.

```bash
sudo arping -c 1 192.168.129.10

```

### 3. Specify a Custom Source MAC Address

Modifies the outgoing ARP frame properties to use a specified custom hardware MAC address (`98:28:a6:04:19:2c`). This is commonly used in administrative simulations to test how switches handle MAC learning updates.

```bash
sudo arping -s 98:28:a6:04:19:2c 192.168.129.10

```

### 4. Direct Specific Interface and Count Bound to Custom MAC

Instructs the utility to send exactly **two ARP requests** out of interface `eth0`, overriding the sender hardware address field with a custom MAC value.

```bash
sudo arping -c 2 -s 98:28:a6:04:19:2c -i eth0 192.168.1.25

```

### 5. Multi-Packet Automated Scan with Explicit Binding

Sends a sequence of **10 ARP requests** out of the `eth0` network interface using a designated source hardware mapping to query target host `192.168.1.31`.

```bash
sudo arping -s 08:00:27:fb:70:09 -c 10 -i eth0 192.168.1.31

```

## Subnet Scan Using `arping` (Bash Script)

This modular script loops through an entire Class-C subnet (`/24`), targeting hosts from `.1` to `.254`. It uses parallel background processing (`&`) to perform a high-speed link-layer discovery sweep:

```bash
#!/bin/bash
# High-speed local subnet layer-2 sweep

SUBNET="192.168.129"

for i in {1..254}; do
    # Target every sequential IP, sending 1 request with a 1-second timeout
    arping -c 1 -w 1 "$SUBNET.$i" 2>/dev/null | grep "reply from" &
done

# Wait for all background parallel tasks to conclude cleanly
wait

```

## Dual-Layer Discovery Script (ICMP + ARP Cross-Check)

This comprehensive discovery script evaluates both **Layer 3 (ICMP)** and **Layer 2 (ARP)** mechanisms. By checking both layers, it successfully identifies standard active nodes *as well as* stealthy, firewalled systems that hide from traditional network pings.

```bash
#!/bin/bash
# Dual-Layer Network Discovery Protocol (ICMP + ARP)

SUBNET="192.168.1"

echo "=================================================="
echo "Starting Dual-Layer Scanning Network Inventory for $SUBNET.0/24"
echo "=================================================="

for i in {1..254}; do
    IP="$SUBNET.$i"
    ONLINE=false
    REASON=""

    # 1. Attempt Layer 3 Verification (ICMP Echo Request)
    ping -c 1 -W 1 "$IP" > /dev/null 2>&1
    if [ $? -eq 0 ]; then
        ONLINE=true
        REASON="Layer 3 (ICMP Response)"
    fi

    # 2. Attempt Layer 2 Verification (ARP Frame Probe)
    arping -c 1 -w 1 "$IP" > /dev/null 2>&1
    if [ $? -eq 0 ]; then
        ONLINE=true
        if [ -n "$REASON" ]; then
            REASON="$REASON & Layer 2 (ARP Response)"
        else
            REASON="Layer 2 (ARP Response - Host is Firewalled)"
        fi
    fi

    # Print out host status if discovered on the network
    if [ "$ONLINE" = true ]; then
        echo "[+] Host Detected: $IP -> [$REASON]"
    fi
done

echo "=================================================="
echo "Scan Complete."
echo "=================================================="

```
