# `hping3` — Advanced TCP/IP Packet Crafter and Network Scanner

**`hping3`** is a powerful command-line network utility designed for crafting, modifying, and transmitting fully customized TCP/IP packets. While standard ping utilities are strictly confined to basic ICMP Echo Requests, `hping3` allows complete structural control over TCP, UDP, ICMP, and RAW-IP protocol frames.

By exposing fine-grained control over protocol flags, byte headers, fragmentation structures, and body payloads, `hping3` serves as a vital toolkit for firewall policy auditing, advanced security reconnaissance, TCP/IP stack fingerprinting, and path latency diagnostics.

---

### Key Features of `hping3`

* **Multi-Protocol Generation:** Crafts custom packet structures for TCP, UDP, ICMP, and Raw IP payloads.
* **Granular Flag Manipulation:** Allows explicit toggling of single or combined TCP header flags (`SYN`, `ACK`, `FIN`, `RST`, `URG`, `PSH`).
* **OS Fingerprinting & Audit Capabilities:** Tests how specific operating systems and security solutions handle anomalous, non-RFC-compliant packet structures (e.g., Christmas scans).
* **Advanced Port Interrogation:** Features a built-in multi-port scanning engine capable of mapping security controls.
* **Network Identity Obfuscation:** Supports arbitrary source IP spoofing and randomized packet origins to simulate distributed conditions.
* **Custom Route Diagnostics:** Includes specialized traceroute modes that calculate round-trip metrics at different network protocol layers.
* **Elevated Privileges Required:** Because it generates raw socket structures bypassing standard system layers, executing `hping3` requires **root privileges** (`sudo`).

# Common `hping3` Usage and Flag Examples

#### Display Help or Usage Information

```bash
hping3 -h
# or
hping3 --help

```

---

#### Protocol Mode Selectors

```bash
# Send RAW IP packets directly to a target destination
sudo hping3 -0 192.168.129.145
# or using long options: sudo hping3 --rawip 192.168.129.145

# Route custom ICMP traffic (similar to ping but customizable)
sudo hping3 -1 192.168.129.145
# or using long options: sudo hping3 --icmp 192.168.129.145

# Target a host using standard UDP datagram packets
sudo hping3 -2 192.168.129.145
# or using long options: sudo hping3 --udp 192.168.129.145

```

---

#### Count, Iteration, and Tuning Controls

```bash
# Enforce a strict execution count to stop after exactly 2 packets
sudo hping3 -c 2 192.168.129.145

# Send exactly two custom ICMP echo requests
sudo hping3 --icmp -c 2 192.168.129.145

# Inject an arbitrary data payload buffer size of 128 bytes into a TCP frame
sudo hping3 192.168.129.145 -d 128

```

---

#### Active Port Scanning Operations

```bash
# Execute a basic TCP port scan across a specified range (Ports 80 through 1000)
sudo hping3 -8 80-1000 192.168.129.145
# or using long options: sudo hping3 --scan 80-1000 192.168.129.145

# Scan selected ports utilizing targeted SYN handshake requests with descriptive logs
sudo hping3 --scan 1-30,70-90 -S 192.168.129.145

# Check boundary ports using a structured FIN-flag scan sequence
sudo hping3 --scan 80,443 -F 192.168.129.145

```

---

#### Identity Obfuscation & Diagnostic Routines

```bash
# Spoof the source IP identity to trick firewalls (Packets appear as if sent from .15)
sudo hping3 -a 192.168.129.15 192.168.129.145

# Force custom ICMP packet formatting with an unassigned structural error code
sudo hping3 -1 -K 4 --force-icmp 192.168.129.145

# Execute an advanced, multi-hop Layer 4 (TCP-based) Traceroute mapping route hops
sudo hping3 -T -S -p 80 8.8.8.8

```

# `hping3` TCP Flags Overview

By altering individual bit states within a 6-bit TCP flag field, you can probe system logic to evaluate stateless firewall rules and mapping filtering strategies.

| Flag Selector | Flag Name | Under-the-Hood Architectural Purpose | Penetration Testing Utility |
| --- | --- | --- | --- |
| `-U` | **URG** (Urgent) | Signals to the receiver that data fields marked by the urgent pointer field override standard stack buffers. | Evaluates firewall behavior and maps non-standard protocol parsing anomalies. |
| `-F` | **FIN** (Finish) | Signals a graceful termination request from the sender, announcing that they have completed their data stream. | Stealth FIN scans can slip past traditional, stateless packet filters. |
| `-S` | **SYN** (Synchronize) | Serves as the initial step during the standard three-way handshake to align transmission sequence numbers. | Identifies open service listener states without completing full system connections. |
| `-R` | **RST** (Reset) | Forces an immediate, abrupt drop of a TCP stream due to an unrecoverable error or dropped connection. | Audits if active network defenses properly log or block reset anomalies. |
| `-P` | **PSH** (Push) | Instructs the receiving network stack to push data up to application layers immediately without buffering. | Measures specific protocol stack behavior and latency tolerances. |
| `-A` | **ACK** (Acknowledge) | Confirms receipt of previous packet blocks by validating the packet's acknowledgment number field. | Maps out static firewall rulesets and determines if ports are filtered or unfiltered. |
| `-X` | **XMAS** (Christmas) | Multi-flag frame where the **FIN**, **PSH**, and **URG** bit states are simultaneously set ("lit up like a Christmas tree"). | Probes target stack compliance with RFC 793; closed ports return a RST, while open systems drop the frame. |

### Example `hping3` Flag-Based Commands

```bash
# Send TCP packets with only the Urgent (URG) bit active
sudo hping3 -U 192.168.129.145

# Send a stealth FIN-flag packet probe
sudo hping3 -F 192.168.129.145

# Initiate a stealth SYN port-availability request
sudo hping3 -S 192.168.129.145

# Send an immediate connection teardown Reset (RST) packet
sudo hping3 -R 192.168.129.145

# Send a Push (PSH) flag packet requesting instant kernel handoff
sudo hping3 -P 192.168.129.145

# Fire an Acknowledgment (ACK) frame to map out state tracking configurations
sudo hping3 -A 192.168.129.145

# Launch an anomalous Christmas (XMAS) scan sequence (FIN+PSH+URG active)
sudo hping3 -X 192.168.129.145

```

### Operational Guardrails

> ⚠️ **Compliance Note:** Because `hping3` allows raw packet generation and high-speed multi-flag streaming, running commands with high-frequency flags (such as `--flood`) can quickly overwhelm network interfaces or degrade system services. Ensure all target boundaries are clearly defined within your active **Rules of Engagement (ROE)** before initiating testing against live networks.
